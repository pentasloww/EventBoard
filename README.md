# EventBoard

EventBoard — вебзастосунок для керування заходами та їх учасниками. Проєкт розроблено мовою Java з використанням Jakarta Servlet та JDBC без застосування готових фреймворків. Основною метою було вивчення принципів побудови багатошарової архітектури вебзастосунків, роботи з HTTP-запитами та взаємодії з базою даних PostgreSQL.

---

##  Функціональні можливості

- перегляд списку заходів;
- перегляд детальної інформації про захід;
- створення нових заходів;
- керування учасниками заходів;
- збереження та отримання даних із бази PostgreSQL;
- обробка HTTP-запитів за допомогою Servlet API.

---

##  Використані технології

- Java 21
- Jakarta Servlet API
- JDBC
- PostgreSQL
- Maven

##  Архітектура проєкту

Проєкт реалізований за багатошаровою архітектурою.

```
Клієнт
   │
Servlet
   │
Service
   │
Repository
   │
PostgreSQL
```

### Структура проєкту

```
src
└── main
    ├── java
    │   ├── model
    │   ├── repository
    │   ├── service
    │   └── servlet
    └── webapp
```

---

##  Опис шарів

### Model

Містить класи предметної області, які описують основні сутності застосунку (наприклад, захід та учасник).

### Repository

Відповідає за взаємодію з базою даних за допомогою JDBC:

- отримання даних;
- додавання записів;
- оновлення інформації;
- видалення даних.

### Service

Містить бізнес-логіку застосунку та забезпечує взаємодію між сервлетами й репозиторіями.

### Servlet

Обробляє HTTP-запити користувачів і формує відповідь, використовуючи бізнес-логіку застосунку.

##  Запуск проєкту

### 1. Клонування репозиторію

```bash
git clone https://github.com/pentasloww/EventBoard.git
```

### 2. Налаштування бази даних

Створіть базу даних PostgreSQL та змініть параметри підключення у класі `DatabaseConnection`.

### 3. Збірка проєкту

```bash
mvn clean package
```

### 4. Запуск застосунку

Розгорніть отриманий WAR-файл на сервері Apache Tomcat.
